# Getting started

This tutorial creates a small contacts list using internal storage.

## 1. Create the storage

1. Open **Components > ContentBuilder NG > Data Storages**.
2. Select **New**.
3. Enter a stable technical name, such as `contacts_cbng`.
4. Enter the title `Contacts`.
5. Select **Internal table** mode.
6. Save.
7. Add `name`, `email`, and `message` fields.
8. Save and run **Datatable Sync** when offered.

The technical name becomes a Joomla-prefixed table. Avoid changing it after
production data exists.

> 📷 *Screenshot to add: creating internal storage and adding fields — `docs/en/img/getting-started-storage.png`*

## 2. Create a view

1. Open **Views**.
2. Select **New**.
3. Enter a clear name, such as `Public contacts`.
4. Select the ContentBuilder NG source.
5. Select the `Contacts` storage.
6. Choose frontend or both contexts.
7. Save once to load the fields.

## 3. Configure fields

In the **View** tab:

- publish the required fields;
- enable **List** for `name` and `email`;
- enable **Search** for `name` and `email`;
- make `name` link to record details;
- make the required fields editable;
- enable **API allowed** only when the API must expose the field.

Then define field order.

## 4. Configure display

In **Advanced options**:

- enable filtering;
- select the initial page size;
- configure sorting;
- enable New and Edit buttons when needed;
- choose whether only published records are visible.

In **Details** and **Edit**:

- start from the examples generated by the selected theme;
- check that an editable template exists;
- keep a simple working version before customization.

## 5. Configure permissions

For an initial registered-user test:

- grant **List Access**, **View**, **New**, and **Edit**;
- grant **Delete** and **Publish** only when required;
- configure frontend permissions, not only backend permissions;
- use own-record permissions when users must manage only their data.

Read [Permissions and ACL](permissions-acl.md) before public deployment.

## 6. Publish a frontend menu item

1. Open Joomla **Menus**.
2. Create a menu item.
3. Select a ContentBuilder NG type: **List View**, **Record Details**, **Create menu
   item** (create/edit), **Latest**, or **Public Views List**.
4. Select the view.
5. Initially keep menu overrides set to their defaults.
6. Publish the menu item.

## 7. Create and edit a record

From the frontend:

1. open the list;
2. select **New**;
3. complete the fields;
4. save;
5. open record details;
6. test **Edit**;
7. verify the return to the list.

If the record is missing, check automatic publication, published-only mode,
language, view permission, ownership, and active filters.
